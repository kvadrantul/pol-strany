# ✅ Решение проблемы с отдачей статики

## Проблема:
404: NOT_FOUND - статические файлы не отдаются на Vercel

## Причина:
В `vercel.json` использовался `routes` вместо `rewrites`, что перехватывало все запросы, включая статические файлы.

## Решение:

### Изменения в `vercel.json`:

**Было (неправильно):**
```json
{
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "api/index.go"
    }
  ]
}
```

**Стало (правильно):**
```json
{
  "rewrites": [
    {
      "source": "/api/(.*)",
      "destination": "/api/index.go"
    }
  ]
}
```

### Разница:

- **`routes`** - перехватывает ВСЕ запросы (включая статику)
- **`rewrites`** - перенаправляет только указанные пути, статика отдается автоматически

### Как теперь работает:

1. **Статические файлы** (`index.html`, `app.js`, `styles.css`, `images/*`) отдаются автоматически из корня (`outputDirectory: "."`)
2. **API запросы** (`/api/*`) перенаправляются через `rewrites` на Go handler
3. **Все остальные запросы** обрабатываются как статика

## Если все еще не работает:

1. Убедитесь, что файлы действительно в корне проекта
2. Проверьте логи деплоя в Vercel Dashboard
3. Убедитесь, что `outputDirectory: "."` указан правильно
4. Передеплойте проект вручную, если нужно
